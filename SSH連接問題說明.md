# SSH 連接問題說明

## 🔴 當前狀態

- ✅ **前端**：正常運行 (http://localhost:3000)
- ❌ **後端**：無法啟動（SSH 連接失敗）

## 🔍 問題診斷

執行測試發現：

```
SSH 主機: 61.216.152.8:22
錯誤: Connection timed out (連接超時)
```

## 💡 可能原因

### 1. 網路連接問題

- 無法連接到遠端伺服器 `61.216.152.8`
- 網路不穩定或斷線

### 2. 需要 VPN

- 遠端伺服器可能需要透過 VPN 才能訪問
- 公司內網限制

### 3. 防火牆阻擋

- 本地防火牆阻擋 SSH 連接
- 遠端防火牆限制

### 4. 伺服器維護

- 遠端伺服器可能正在維護
- SSH 服務未啟動

## 🔧 解決方案

### 方案 1：檢查網路連接

```bash
# 測試是否能 ping 通伺服器
ping 61.216.152.8

# 測試 SSH 端口是否開放
telnet 61.216.152.8 22
```

### 方案 2：連接 VPN

如果伺服器在公司內網：

1. 連接公司 VPN
2. 確認 VPN 連接成功
3. 重新啟動後端

### 方案 3：檢查防火牆

1. 檢查 Windows 防火牆設置
2. 確認允許 Python 程序訪問網路
3. 檢查防毒軟體是否阻擋

### 方案 4：使用本地資料庫（開發用）

如果只是測試，可以：

1. 安裝本地 PostgreSQL
2. 匯入測試資料
3. 修改配置使用本地資料庫

## 💬 常見問題

### Q: 為什麼前端能啟動但後端不行？

A: 前端是純靜態頁面，不需要資料庫。後端需要透過 SSH 連接到遠端資料庫。

### Q: 可以不用 SSH 嗎？

A: 目前系統設計需要 SSH 隧道連接到遠端 PostgreSQL 資料庫。

### Q: 如何確認是否需要 VPN？

A: 嘗試 ping 伺服器 IP，如果無法 ping 通，可能需要 VPN。

### Q: 有沒有離線測試方案？

A: 可以設置本地 PostgreSQL 資料庫並匯入測試資料。
